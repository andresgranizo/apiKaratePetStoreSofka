Por que Karate?
Basicamente porque no necesitas escribir step definitions aparte como en Cucumber normal.
Todo va directo en el .feature y eso ahorra bastante tiempo. Ademas los matchers
para JSON son muy buenos, tipo "match response contains { name: '#string' }" y ya.

Que hice
--------
Implemente el CRUD completo de mascotas:
1. Crear una mascota
2. Consultarla por ID
3. Actualizarla (nombre + status "sold")
4. Buscar por status

Lo mas complicado fue lo de los IDs. Al principio puse IDs fijos (tipo 999)
y a veces fallaba porque ya existia esa mascota en la API. La solucion fue
generar IDs random con Math.random() * 100000.

Otro tema fue que queria encadenar los scenarios (que el GET use el ID del POST)
pero Karate ejecuta cada scenario independiente. Termine haciendo que cada uno
cree su propia data y listo.

Lo bueno de Karate
- No escribes Java para los tests, solo el runner
- Los reportes se generan solos y quedan bastante bien
- La sintaxis para validar responses es muy directa

Que falta
- Tests de error (mandar datos malos, IDs que no existen, etc)
- Probar los otros endpoints de Store y User
- Meter autenticacion con API key

Me tomo unas 3-4 horas armar todo y dejarlo corriendo bien.
